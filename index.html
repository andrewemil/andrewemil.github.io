Acid Server: 1 Walkthrough

Aug 22nd, 2015 3:58 pm

Acid Server is a boot2root challenge built by Avinash and hosted by vulnhub. The challenge is completely focused on web exploitation and the end goal is to escalate the privileges to root and capture the flag. This walkthrough shows how I solved the challenge so let the fun begin!

After booting Acid Server and Kali on virtual box (Bridged Adapter), I used netdiscover to find the ip address of Acid server:

So, Acid’t ip address is 192.168.2.102 and our kalis ip address is 192.168.2.108 as shown below:

Nmap

Lets find what services are running on Acid using nmap:

Nmap found that apache web server was running on port 33447,so lets visit the main index page for the web service http://192.168.2.102:33447/

It seems that there isn’t anything useful here, so my next destination was dirbuster to find any hidden directories or files that might be interesting:

Note:I used directory-list-2.3-medium.txt wordlist which is located in /usr/share/dirbuster/wordlist/

So, Dirbuster found a new directory named “Challenge”, and the index page “http://192.168.2.102:33447/Challenge/js/index.js” of this directory is a login form as we can see:

After searching for the login script in google using keywords “secure login peredur.net”, I found its main github repository and the default dabtabase is configured with a default user “test_user” as you can see in the description of the github repo:

At this point, I tried to get RCE using the LFI vulnerability by using apache log poisining, reading /proc/self/environ, or any logs in the file descriptor. But it seems the apache user doesn’t have any read access on those files and only php://filter wrappter worked which is very useful to further enumerate the php files There is only one readable file descriptor /proc/self/fd/14 but I found later after examining the files that it includes only php sessions which weren'y controlable by any mean so it was usless.
